# Пояснение к Этапу №3 \| AI/ML \| Парсинг данных

## Проект: TeachAI - интеллектуальная обучающая система

------------------------------------------------------------------------

## 1. Архитектурная постановка задачи

Проект TeachAI представляет собой интеллектуальную обучающую систему, основанную на использовании готовой предобученной GPT-модели через API OpenAI.

Система может составлять план обучения, писать уроки, давать примеры кода, создавать тесты и отвечать на вопросы студентов.

Система:

- не обучает модель с нуля;

- не выполняет дообучение (fine-tuning);

- не использует собственный обучающий датасет;

- не решает задачи классификации, регрессии, сегментации или object detection.

Основной механизм работы системы --- использование знаний готовой предобученной языковой модели с динамической генерацией контента на основе пользовательских запросов и параметров профиля обучающегося.

------------------------------------------------------------------------

## 2. Связь с Этапом №2 (Сбор базы данных)

На предыдущем этапе было обосновано и принято решение, что:

- сбор базы данных для обучения нейросети не требуется;
- дообучение модели не производится;
- используется готовая предобученная GPT-модель;
- генерация осуществляется в режиме API-запросов.

Таким образом, в архитектуре проекта отсутствует:

- обучающая выборка;

- размеченные данные;

- локальная база изображений или текстов;

- пайплайн подготовки датасета.

------------------------------------------------------------------------

## 3. Анализ необходимости парсинга данных

Парсинг данных требуется в следующих случаях:

1. При формировании обучающей выборки.
2. При автоматическом сборе данных с сайтов.
3. При подготовке размеченных данных для ML-задач.
4. При построении DataFrame / Excel-таблиц для обучения модели.
5. При подготовке изображений и аннотаций (классификация, сегментация, detection).

В рамках проекта TeachAI для готовой предобученной языковой модели:

- отсутствует обучающий процесс;
- отсутствует локальный датасет;
- отсутствует задача разметки;
- отсутствует необходимость структурирования внешних данных для обучения модели;
- система не извлекает данные с внешних сайтов для последующего обучения.

Генерация учебного контента выполняется непосредственно языковой моделью в ответ на сформированный промпт.

Следовательно, парсинг данных как этап подготовки обучающей выборки архитектурно не предусмотрен.

------------------------------------------------------------------------

## 4. Тип задачи проекта

Проект относится к категории:

> AI-система прикладного уровня, использующая знания предобученной LLM-модели для генерации персонализированного образовательного контента.

Это не задача классификации, регрессии, сегментации, object detection. Это - не обучение собственной нейросети.

Поэтому форматы хранения данных, приведённые в описании задания (структурирование по папкам, маски сегментации, bounding boxes и т.д.),
к данному проекту не применимы.

------------------------------------------------------------------------

## 5. Вывод

Поскольку:

- проект не предполагает обучение или дообучение модели;
- отсутствует необходимость формирования датасета;
- отсутствует сбор данных из внешних источников;
- отсутствует подготовка структурированных обучающих данных;

этап «Парсинг данных» в рамках проекта TeachAI не требуется.

Архитектура проекта изначально построена как inference-based система, использующая возможности уже обученной языковой модели.
